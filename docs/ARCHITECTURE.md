# Architecture

## Product Shape

Reef is a small declarative cluster deployment system. The operator maintains environment variables and one root seed; Reef derives the full cluster from those inputs, renders deployment artifacts, applies them to remote nodes, and generates subscription files.

The first-phase product shape:

- Use `just` as the only user entry point.
- Use Python for configuration parsing, secret derivation, and file generation.
- Use Ansible for remote deployment, upgrade, and deletion.
- Use Next.js on Vercel to serve private subscription URLs.
- Use Docker-compatible Compose to run the local four-node integration test.

## Managed Node Layout

Reef manages only a fixed remote directory and the service files it installs:

```text
/opt/reef/
в”њв”Ђв”Ђ bin/
в”њв”Ђв”Ђ certs/
в”њв”Ђв”Ђ config/
в””в”Ђв”Ђ .managed.json
```

`/opt/reef/.managed.json` is a non-secret marker. It records only ownership, version, node identity, managed file lists, and render hashes. It must not contain the root seed, node passwords, subscription tokens, or private keys.

System services are declared by the provider manifest. Ansible installs, starts, restarts, and deletes them.

## Test Layout

`just test` uses four gateway nodes and one internal echo target:

```text
entries: sg, jp
exits:   us, uk
```

Test containers run real `systemd` and `sshd` without `privileged: true`:

```yaml
cap_add:
  - SYS_ADMIN
tmpfs:
  - /run
  - /run/lock
```

Test-only SSH address mappings live inside `tests/` and are not part of the production configuration model.
